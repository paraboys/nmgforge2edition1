# PulseDesk Sprint Backlog

This document outlines the high-level roadmap and backlog for building the PulseDesk SaaS. The project is divided into 5 tight, independently deliverable sprints.

## Sprint 1: Core Foundation & Auth
**Focus:** Project setup verification, Multi-tenancy, Organizations, Authentication, Roles, Sanctum.
**Goal:** Establish the secure, tenant-isolated foundation so that users can log in and see only their organization's data.

## Sprint 2: Ticket Engine
**Focus:** Ticket CRUD, Assignment, Priority, Status, Tags.
**Goal:** Allow users to create, view, and manage support tickets with basic metadata.

## Sprint 3: Communication Layer
**Focus:** Conversations, Internal Notes, Public Replies, Attachments architecture.
**Goal:** Enable back-and-forth communication on tickets, separating public customer replies from internal agent notes.

## Sprint 4: Insights & Management
**Focus:** Search, Filters, Dashboard, Metrics, Activity Log.
**Goal:** Provide visibility into the ticket queue, allow easy finding of tickets, and track the audit trail of actions.

## Sprint 5: Quality, Polish & Delivery
**Focus:** Tests, Seeders, GitHub Actions, Documentation, Deployment, Submission checklist.
**Goal:** Finalize the build with robust seeders (for demo data), automated CI tests, and submission preparation.
